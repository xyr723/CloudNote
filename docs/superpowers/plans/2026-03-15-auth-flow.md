# Auth Flow Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `App.tsx` 中的认证导航、session 恢复和用户状态管理下沉到 `src/features/auth/**`，让 `App.tsx` 只保留主题状态和 app shell。

**Architecture:** 新增 `AuthFlow` 作为 auth feature 的入口编排组件，内部持有 `authProvider`、登录态恢复、登录注册切换和登出回退。`App.tsx` 继续管理 `themeColor` / `isDarkMode` 并包裹 `NavigationContainer`，`HomeScreen` 只接收当前用户、用户回写和 `onSignOut` 回调，不再直接接触 auth provider。

**Tech Stack:** React Native、React Navigation、React hooks、Jest、react-test-renderer、TypeScript
